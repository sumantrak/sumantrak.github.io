Designing Bandit Objective Functions

The core of any bandit algorithm is the design of its objective function — this governs how the learner balances exploration and exploitation. Most classic objectives are cast as forms of regret minimization, but modern applications demand broader criteria.

🔧 General Design Principles

Objective functions are typically crafted based on:

  • Information structure: What kind of feedback do we get (numeric, ordinal, full reward vector, preference comparisons)?
  • Decision horizon: Are we optimizing over a fixed number of rounds or indefinitely?
  • Application goals: Maximize total reward? Identify the best arm? Ensure fairness? Minimize risk?

🎯 Classical Objectives

Objective Definition Use Case
Cumulative Regret $R_T = T \mu^* - \sum_{t=1}^T \mathbb{E}[\mu_{a_t}]$ Long-term revenue, ad placement
Simple Regret $R = \mu^* - \mathbb{E}[\mu_{\hat{a}}]$, where $\hat{a}$ is the selected best arm Pure exploration, scientific discovery
PAC (Probably Approximately Correct) Find an arm within $\epsilon$ of optimal with probability $\ge 1 - \delta$ Safe decision-making, high-stakes domains

⚖️ Modern and Domain-specific Objectives

Objective Description Domains / Examples
Risk-sensitive regret Penalize high variance or downside loss Finance, healthcare
Fairness-aware regret Equal opportunity or demographic parity across arms Hiring, lending
Budgeted objectives Maximize reward under cost/resource constraints Mobile devices, sensor networks
Multi-objective trade-offs Optimize several competing metrics simultaneously Recommendation systems
Robust regret Worst-case regret under model misspecification or adversarial reward shifts Safety-critical applications
Information gain Maximize knowledge about arm means (e.g. Thompson Sampling variants) Adaptive experimentation

Objective Functions by Bandit Type

We now briefly connect different bandit types with typical objectives and design constraints.

Bandit Type Common Objective Function(s) Comments
Stochastic MAB Cumulative regret UCB, TS, ε-greedy apply
Best-arm Identification Simple regret / PAC Focused on exploration
Combinatorial Bandits Submodular reward or structured regret Require arm composition models
Contextual Bandits Contextual regret Linear, neural reward functions
Dueling / Ordinal Bandits Preference-based regret Partial feedback, pairwise comparisons
Causal Bandits Interventional regret Causal graphs, do-calculus guided
Multi-objective Bandits Pareto regret, weighted-sum regret Trade-offs in reward dimensions
Adversarial Bandits Regret vs worst-case comparator EXP3, minimax approaches
Batch Bandits Batch regret, regret per round Limits adaptivity
Private Bandits Regret under privacy constraints Adds noise, affects confidence bounds
Risk-aware Bandits CVaR or tail-risk regret Tailor exploration to risk profile
Principle Idea Examples
Optimism in the Face of Uncertainty Prefer arms with uncertain but potentially high rewards UCB, LinUCB
Bayesian Reasoning Maintain and update belief distributions over rewards Thompson Sampling, Bayesian UCB
Minimax / Adversarial Framing Prepare for the worst-case reward scenario EXP3, Robust UCB
Information-Directed Sampling Balance information gain with expected regret IDS, Bootstrap TS
Structural Exploitation Leverage known correlations or constraints (linearity, graph, etc.) LinTS, KernelUCB, Graph Bandits
Exploration Safety / Risk Aversion Avoid risky arms unless confidence improves CVaR bandits, Safe-UCB
Personalization / Adaptation Adapt to user/task/environment specifics Meta-bandits, Contextual TS

Algorithm BanditLearner:

1. Initialize:
    - Define the arm space A
    - Set priors OR confidence bounds for each arm a ∈ A
    - If structured: set up model assumptions (e.g., linear, Gaussian process, graphs)
    - If contextual: prepare feature extractor φ(x)

2. For t = 1 to T do:
    a. Observe context x_t (if contextual)
    
    b. Estimate:
        - Use data/history to update:
            • Posterior distribution (Bayesian)
            • Confidence intervals (Frequentist)
            • Preference/utility models (Ordinal)
        - Calculate scores:
            • UCB = estimate + bonus
            • TS: sample reward from posterior
            • EXP3: compute softmax-weighted probabilities
            
    c. Select arm a_t:
        - Use score-based selection:
            • Greedy / ε-greedy / Sampling
            • Top-k (combinatorial) / argmax

    d. Play arm a_t and receive feedback:
        - Full reward, partial feedback, ordinal, delayed, etc.

    e. Update:
        - Incorporate new feedback into:
            • Belief distributions
            • Empirical estimates
            • Model parameters
            
    f. [Optional] Constraints / Meta-Adjustments:
        - Budget checks
        - Privacy noise injection
        - Batched update logic
        - Risk thresholds

3. End
